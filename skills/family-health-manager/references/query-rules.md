# Query Rules

## 适用场景

当用户提出以下问题时，使用本规则：
- 现在或以前在吃什么药
- 某项检查结果如何
- 某个指标最近趋势如何
- 某次门诊、住院、体检发生了什么
- 某份报告在哪里
- 某位成员有哪些长期背景、过敏史或重要既往史

## 信息来源优先级

回答时优先使用以下顺序：

1. `records/` 中的原始文件
2. tracker 中的结构化记录
3. 对应事件目录下的 `summary.md`
4. 成员 `profile.md`

如果不同来源冲突，不要猜测；明确指出冲突并请用户确认。

## 通用查询流程

1. 明确目标成员
2. 明确问题范围
3. 读取最相关的数据源
4. 需要时补读 `summary.md` 或原始文件
5. 先回答结论，再给证据，再说不确定性


回答输出格式默认使用论文引用式编号：
- 在正文证据处使用 `[1]`、`[2]`、`[3]`
- 在答案最后单独列出 `引用来源`
- 每条 `引用来源` 至少包含：日期、`来源文件`、`归档路径` 或事件目录
- 若同一结论由多个来源共同支持，可连续引用多个编号，如 `[1][2]`

推荐结构：

```md
## 结论
...

## 证据
- ...
- ...

## 冲突或不确定性
...

## 引用来源
[1] 2026-03-12，来源文件：records/.../甲功五项.pdf；归档路径：records/.../2026-03-12-内分泌复诊/
[2] 2026-03-12，来源文件：records/.../summary.md；归档路径：records/.../2026-03-12-内分泌复诊/
```

## 药物问题

优先读取：
- `medications`
- 相关事件的 `summary.md`
- 原始处方、出院带药、医嘱文件

回答时：
- 先说当前可确认的用药或历史用药
- 再给开始 / 结束时间、状态、用法用量
- 再附日期和 `来源文件`

不要把“曾被开具”直接说成“现在仍在使用”。

## 检查结果与趋势问题

优先读取：
- `measurements`
- 相关化验单、影像报告、体检报告
- 必要时读 `summary.md`

当用户询问“某项检查的时间序列变化”“某个指标历次结果”“近几年某项检查趋势”时：
- 以 `measurements` 作为主数据源
- `timeline` 仅用于辅助定位相关事件、复查上下文和归档路径
- 不要为了项目级检索而把 `timeline` 当作主要数值事实来源

回答时：
- 先给结论
- 再按时间列关键结果
- 保留原始单位
- 如单位或参考范围不一致，明确提示
- 如果相关资料来自多页报告，且当前仅完成部分页复核，只有在未复核部分可能影响结论时，才明确说明当前结论基于已复核页面

不要把数值变化直接上升为医学诊断结论。

## 事件回溯问题

优先读取：
- `timeline`
- 对应事件目录下的 `summary.md`
- 必要时补读原始文件

回答时：
- 先指出是哪次事件
- 用 2 到 5 句话概括重点
- 尽量附 `归档路径` 和关键 `来源文件`

## 报告定位问题

优先读取：
- `timeline`
- 对应年份下的 `records/`
- 对应 `summary.md`

回答时：
- 先给事件目录或 `归档路径`
- 再给最可能的 `来源文件`
- 如不能完全确认，明确说明不确定性

## 长期背景总结问题

优先读取：
- `profile.md`
- `timeline`
- 与问题最相关的近期 `summary.md`
- 必要时查看 `medications` 或 `measurements`

回答时：
- 先列 3 到 6 条最重要背景
- 再列关键药物、既往史、过敏史
- 明确这是档案整理，不替代医生判断

## 冲突与不确定性处理

当出现以下情况时，不要自行拍板：
- 原始文件与 tracker 不一致
- 日期不一致
- 药物状态不清
- OCR 文本模糊
- 同一指标名称或单位存在歧义

推荐表达：
- “这里有来源冲突：……”
- “原始报告写的是……，tracker 当前记录为……”
- “现有档案只能确认到……，更进一步需要你确认或查看原件”

## 常见检查项目规范称呼

命名规则与示例见 `references/data-model.md` measurements 一节。

## 回答风格

- 使用简洁中文回答
- 必要时保留常见 English medical terms，如 `TSH`、`CT`、`MRI`、`HbA1c`
- 先给结论，再给证据
- 尽量附日期、`来源文件`、`归档路径`
- 不把档案查询扩展成医学诊断

如果用户的问题明显是紧急医疗判断，可先说明档案中的事实，再建议尽快联系医生或急诊。
